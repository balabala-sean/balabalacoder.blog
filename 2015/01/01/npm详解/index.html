2、npm的安装和更新

在安装nodejs的时候，是默认帮安装了npm的，所以此步略过。详情可以参考nodejs的安装。但是有一个坑：

  • 为什么是sudo管理员权限执行？因为npm的默认保存package的目录是在/usr/local下的，所以一般当前用户是没有写的权限的，只有读权限
  • 为什么要-g呢？-g表示全局安装，如果没有-g，执行npm install的时候会在当前目录(一般也是项目目录)下创建一个node_modules的目录，并把要install的package存放在这个目录下，但是这些installed package只能应用于当前的应用程序。如果是-g，则会在上面说的/usr/local下全局进行安装，就是所有的应用程序都可以进行require来加载package。

4、package.json

如上所述，npm install的时候如果不加-g，则会在当前目录下创建一个名为node_modules的文件夹，用来保存第三方的依赖，所以，我们如期的看到了这个node_modules的目录，然后这个目录下面有什么呢：

哦，这个下面有个jquery,再回顾下我们的package.json文件，其中有个dependencies属性，里面包含了jquery和它的版本说明。看到这里，我们不难联想，npm在执行install的时候，会默认的去读取package.json文件，然后根据里面的dependencies属性，为我们自动的下载所需要的依赖。值得一提的是，我们的jquery版本指定的是”>=1.4.3”，而npm在没有读到准确版本的时候，会自动下载最新的版本(如果换成1.4.3，则只下载jquery-1.4.3的版本)。

ok，第一个问题解决了，再说第二个问题：package.json文件中的各个属性有什么约束或者限制么：

name: defaults to author name unless in a git directory, in which case it will be the name of the repository
version: always 1.0.0
main: always index.js
scripts: by default creates a empty test script
keywords: empty
author: whatever you provided the CLI
license: ISC
repository: will pull in info from the current directory, if present
bugs: will pull in info from the current directory, if present
homepage: will pull in info from the current directory, if present

总共是这些属性，并不是所有的属性在package.json文件中都是必须的，其实我们刚才上面用到的3个，就是最常见最关键的三个。如果需要别的属性另行查阅资料或者Google搜一下。

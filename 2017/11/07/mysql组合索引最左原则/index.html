mysql组合索引最左原则

发表于 2017-11-07 | 分类于 mysql

在一些查询条件较多的时候，我们往往会选择组合索引，类似：

需要注意的是

  • 索引的字段需要按照辨识度(区分度)排序，比方column_a, column_b, column_c这样的组合索引，那么column_a就是辨识度较高的字段，要放在组合索引的最左边。

  • 组合索引在where条件中出现的顺序是无关的，会被sql查询优化器重新排序。比方where A and B and C， 不管A,B,C在sql书写上的顺序如何，都会被SQL查询优化器进行重新排序后再执行。

并发的3个特性

发表于 2016-10-07 | 分类于 并发

在并发编程中，通常会遇到以下三个问题：原子性、可见性、有序性。

1.原子性

原子性：即一个操作或者多个操作 要么全部执行并且执行的过程不会被任何因素打断，要么就都不执行。

一个很经典的例子就是银行账户转账问题：

比如从账户A向账户B转1000元，那么必然包括2个操作：从账户A减去1000元，往账户B加上1000元。

试想一下，如果这2个操作不具备原子性，会造成什么样的后果。假如从账户A减去1000元之后，操作突然中止。然后又从B取出了500元，取出500元之后，再执行 往账户B加上1000元 的操作。这样就会导致账户A虽然减去了1000元，但是账户B没有收到这个转过来的1000元。

所以这2个操作必须要具备原子性才能保证不出现一些意外的问题。

同样地反映到并发编程中会出现什么结果呢？

举个最简单的例子，大家想一下假如为一个32位的变量赋值过程不具备原子性的话，会发生什么后果？

假若一个线程执行到这个语句时，我暂且假设为一个32位的变量赋值包括两个过程：为低16位赋值，为高16位赋值。

那么就可能发生一种情况：当将低16位数值写入之后，突然被中断，而此时又有一个线程去读取i的值，那么读取到的就是错误的数据。

2.可见性

可见性是指当多个线程访问同一个变量时，一个线程修改了这个变量的值，其他线程能够立即看得到修改的值。

此时线程2执行 j = i，它会先去主存读取i的值并加载到CPU2的缓存当中，注意此时内存当中i的值还是0，那么就会使得j的值为0，而不是10.

这就是可见性问题，线程1对变量i修改了之后，线程2没有立即看到线程1修改的值。

并发中的可见性

发表于 2016-09-17 | 分类于 并发

在Java的多线程模型中，线程之间是不允许直接传递数据进行交互的，它们之间的交互只能通过共享变量来实现。当创建一个对象实例的时候，它是被分配在堆(heap)上的，每个线程运行时都有自己的工作内存(stack), 当线程操作这个对象的时候，首先从堆中创建一个此对象的引用副本到自己的工作内存中，线程运行完毕后，再把工作内存中的对象副本刷新到堆中。在多线程的情况下，即一个共享对象存在了多个副本(在不同的线程栈中)，如果一个线程修改了此对象，那么别的线程就能看到被修改后的值了。

volatile详解

发表于 2016-08-07 | 分类于 并发

在java的线程中由于线程栈的存在，对于变量的每一次读写，并不是从主内存区(heap)中获取最新的值，这就可能造成线程A、线程B同时读取了某一个变量x的值，然后线程A对x做了修改，而线程B还在用线程栈中的x值，造成了最终的数据不一致问题。

volatile译为“不稳定的”，这就指示jvm，线程A、线程B每次使用x的时候，都去主内存中进行读取。这样在任何时候，多个线程之间看到的共享变量都是同一个值。内部的实现是禁用了高速缓存策略（cpu cache）
